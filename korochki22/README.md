# Корочки.есть

Готовый учебный проект портала записи на онлайн-курсы ДПО.

## 1. Выбранный стек

- **Backend:** чистый PHP 7.1+ в MVC-структуре без фейковой логики
- **Database:** MySQL / MariaDB через PDO
- **Frontend:** HTML5, CSS3, нативный JavaScript
- **Архитектура:** Front Controller + Router + Controllers + Models + Views
- **Безопасность:** password_hash/password_verify, CSRF-защита, экранирование вывода, защита роутов по ролям
- **UI:** адаптивный интерфейс, отдельная мобильная проработка под 390x844

> Проект сделан без зависимости от Composer и Node, чтобы его можно было быстро запустить в OpenServer / XAMPP / Denwer-подобной среде.

## 3. Карта страниц

- `/` — главная страница + слайдер
- `/register` — регистрация
- `/login` — авторизация
- `/applications/create` — создание заявки
- `/applications` — мои заявки
- `/reviews/create?application_id=ID` — форма отзыва
- `/admin/applications` — панель администратора

## 4. Быстрый запуск

1. Скопируйте папку проекта в папку доменов вашего локального сервера.
2. Создайте пустую базу данных, например `korochki_est`.
3. Откройте файл `config/database.php` и укажите:
   - host
   - port
   - database
   - username
   - password
4. Откройте проект в браузере.

При первом запуске приложение **само создаст таблицы** и добавит администратора.

## 5. Данные администратора

Из-за противоречия в ТЗ поддержаны **оба** варианта входа администратора:

- `admin / admin`
- `Admin / KorokNET`

Оба варианта ведут в одну и ту же админ-учётную запись с ролью `admin`.

## 6. Демо-данные

По умолчанию создаётся только администратор.  
При желании можно включить демо-наполнение в `config/app.php`:

```php
'seed_demo_data' => true,
```

Тогда автоматически добавятся:
- 2 тестовых пользователя
- несколько заявок
- пример завершённой заявки с отзывом

## 7. Валидация

Реализовано:

- уникальность логина
- логин только латиница и цифры, минимум 6 символов
- пароль минимум 8 символов
- ФИО только кириллица и пробелы
- телефон строго `8(XXX)XXX-XX-XX`
- email в корректном формате
- проверка даты в формате `ДД.ММ.ГГГГ`
- запрет отзыва не по своей заявке
- запрет отзыва до статуса `Обучение завершено`
- один отзыв на одну заявку
- защита страниц по роли
- проверка допустимых статусов и способов оплаты
- CSRF для всех POST-форм
